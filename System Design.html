letsSplitIt

System design — architecture, data model, and core flows
Stack
Spring Boot 4 / Java 21
Datastore
PostgreSQL · JPA
Auth
JWT · BCrypt
Style
Layered REST

letsSplitIt is a stateless REST backend for tracking shared expenses between friends and groups. It splits bills four ways (equal, exact, %, shares) via a strategy registry, maintains running pair-balances, supports group debt simplification, and emits domain events that fan out to email notifications.

05

Split strategies

Four interchangeable algorithms behind one interface. New split modes drop in as a Spring bean — the registry auto-discovers them.

Behaviour at a glance

Given a total of ₹1200 and 3 participants, each strategy distributes the burden differently. The payer's share is implied — they are charged the balance after SharedExpense rows are written.

SplitType Input shape Result for ₹1200 / 3
EQUAL just userIds 400 · 400 · 400
EXACT amount each 500 · 400 · 300
PERCENTAGE % each (Σ = 100) 50% → 600 · 30% → 360 · 20% → 240
SHARES share weight 2 : 1 : 1 → 600 · 300 · 300
EXTENSION POINT
Adding a 5th strategy
  1. Add ITEMIZED (or whatever) to Expense.SplitType.
  2. Implement SplitStrategy with @Component.
  3. Done. The SplitStrategyRegistry picks it up by type() at startup.
